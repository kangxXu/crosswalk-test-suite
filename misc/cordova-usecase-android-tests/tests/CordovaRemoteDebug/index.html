Pre-condition

  • Connect device to host with USB cable
  • The Chrome version must be >= Cordova Crosswalk version

Test Steps

  1. Run the following commands:
    • $ unzip crosswalk-cordova-<version>-arm.zip
    • $ cd /localpath/crosswalk-cordova-<version>-arm
    • $ ./bin/create remotedebugging com.example.remotedebugging RD --shared
    • $ cd remotedebugging
    • $ ./cordova/build --debug
    • $ ./cordova/run
  2. Open 'chrome://inspect' on host chrome browser
  3. Click 'inspect' button
  4. Click 'Sources' option
  5. Add a breakpoint
  6. Click "elements" button
  7. Select one html code then click the right mouse button
  8. Select 'Edit as HTML', then edit the code 'add or remove text'
  9. Check the page

Purpose:

Verify the Cordova package can remote debug successfully

Expected Results:

  1. Able to create 'mobilespec-debug.apk' under bin folder and launch the app successfully
  2. The 'cordova_mobile_spec' app name display in the inspection page
  3. The 'cordova_mobile_spec' app source code display in page
  4. Display Debug information, such as 'Breakpoints' 'DOM Breakpoints'
  5. The Breakpoint can add successfully
  6. The button can work well
  7. Display shortcut menu
  8. The source code could be changed
  9. The change takes effect
